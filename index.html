1 Gambaran umum dan alur kerja

Sebelum mulai, penting untuk memahami bagaimana sistem ini bekerja dari awal sampai akhir.

Apa itu WhatsApp Business API?

WhatsApp Business API adalah layanan resmi yang disediakan oleh Meta (perusahaan induk WhatsApp) untuk mengirim dan menerima pesan WhatsApp melalui kode program. Berbeda dengan library tidak resmi seperti Baileys yang rawan dibanned, API ini dijamin stabil karena memang disediakan oleh pembuat WhatsApp sendiri.

Alur kerja secara keseluruhan

Berikut urutan yang akan kamu lakukan dalam tutorial ini:

  1. Siapkan bahan — akun Facebook, nomor HP khusus, dan komputer
  2. Daftar di Meta Developer — buat akun developer dan buat "App"
  3. Aktifkan WhatsApp — tambahkan WhatsApp sebagai produk di App tersebut
  4. Catat credentials — Phone Number ID dan Access Token
  5. Daftarkan penerima — tambahkan nomor WA yang akan menerima pesan
  6. Test kirim pesan — kirim pesan pertama via dashboard Meta atau cURL
  7. Otomasi dengan Node.js — buat script sederhana untuk kirim pesan otomatis
  8. Buat token permanen — agar tidak perlu ganti token setiap 24 jam
Keterbatasan yang perlu diketahui WhatsApp Business API hanya bisa mengirim pesan personal (satu-ke-satu). API ini tidak bisa mengirim pesan ke grup WhatsApp dan tidak bisa melakukan tag all. Semua pesan yang dikirim harus menggunakan format "template" yang sudah disetujui oleh Meta terlebih dahulu.

3 Apakah benar gratis?

Ya. Meta memberikan kuota 1.000 pesan gratis setiap bulan tanpa syarat apapun.

1.000 pesan gratis per bulan
Berlaku untuk kategori utility. Tanpa kartu kredit. Reset otomatis setiap awal bulan.

Jika penggunaan melebihi 1.000 pesan, biaya tambahannya adalah sekitar Rp 300-350 per pesan untuk kategori utility (notifikasi, alert). Untuk konteks: jika kamu mengirim 5 pesan per hari ke 3 orang selama sebulan penuh, totalnya hanya 450 pesan — masih jauh di bawah kuota gratis.

4 Daftar akun Meta Developer

Langkah pertama adalah membuat akun di portal developer Meta dan membuat sebuah "App".

Buka portal Meta Developer

Buka browser dan kunjungi developers.facebook.com. Klik tombol "Get Started" di pojok kanan atas halaman. Kamu akan diminta login menggunakan akun Facebook. Jika belum punya, buat akun Facebook terlebih dahulu.

Lengkapi registrasi developer

Setelah login, Meta akan menampilkan wizard registrasi. Kamu perlu menyetujui Terms of Service, memverifikasi alamat email, dan memilih role sebagai "Developer". Proses ini hanya perlu dilakukan sekali.

Buat App baru

Setelah registrasi selesai, kamu akan masuk ke dashboard. Lakukan langkah berikut:

  1. Klik menu My Apps di bagian atas halaman
  2. Klik tombol Create App
  3. Di halaman "What do you want your app to do?", pilih Other, lalu klik Next
  4. Di halaman "Select an app type", pilih Business, lalu klik Next
  5. Isi formulir:
    • App name: bebas, contoh "WA Bot" atau "Server Alert"
    • App contact email: alamat email kamu
    • Business Account: pilih yang sudah ada, atau pilih "I don't want to connect a business portfolio yet"
  6. Klik Create App

Buat Meta Business Account (jika diminta)

Beberapa fitur WhatsApp API memerlukan Business Account. Jika kamu belum punya:

  1. Buka business.facebook.com di tab baru
  2. Klik Create Account
  3. Isi nama bisnis (boleh nama apapun), nama kamu, dan alamat email
  4. Klik Submit

Setelah selesai, kembali ke halaman App dan hubungkan dengan Business Account yang baru dibuat.

5 Aktifkan WhatsApp di dashboard

Setelah App dibuat, tambahkan WhatsApp sebagai produk dan dapatkan credentials untuk API.

Tambahkan produk WhatsApp

Di halaman dashboard App kamu, scroll ke bawah sampai menemukan daftar produk. Cari WhatsApp dan klik tombol Set Up di sebelahnya. Pilih Meta Business Account yang sudah dibuat, lalu klik Continue.

Catat tiga informasi penting

Setelah WhatsApp diaktifkan, kamu akan diarahkan ke halaman API Setup. Di halaman ini terdapat tiga informasi yang harus kamu catat dan simpan:

Simpan ketiga nilai ini
Phone Number ID : 123456789012345      (untuk memanggil API)
WABA ID         : 987654321098765      (untuk manajemen akun)
Access Token    : EAAGk0B3a...         (untuk autentikasi, berlaku 24 jam)

Phone Number ID dan Access Token adalah dua nilai yang paling sering digunakan. WABA ID biasanya hanya dibutuhkan untuk manajemen tingkat lanjut.

Token dari halaman ini hanya berlaku 24 jam. Untuk penggunaan jangka panjang, kamu perlu membuat token permanen. Caranya ada di Step 10.

6 Daftarkan nomor penerima

Selama akun masih dalam mode testing, kamu hanya bisa mengirim pesan ke nomor yang sudah didaftarkan secara manual.

Tambahkan nomor penerima

  1. Di halaman API Setup, cari bagian bertuliskan To
  2. Klik Manage phone number list
  3. Klik Add phone number
  4. Masukkan nomor WA penerima dengan format kode negara tanpa tanda plus. Contoh: 6281234567890
  5. WhatsApp akan mengirimkan kode verifikasi ke nomor tersebut. Minta penerima untuk memberikan kode itu, lalu masukkan di halaman dashboard

Tentang format nomor

Format nomor yang benar dan salah:

Format nomor
BENAR : 6281234567890   (kode negara 62 + nomor tanpa 0 di depan)
SALAH : +6281234567890  (jangan pakai tanda +)
SALAH : 081234567890    (jangan pakai 0 di depan)
SALAH : 81234567890     (harus ada kode negara 62)

7 Buat message template

WhatsApp Business API mengharuskan semua pesan dikirim dalam format template yang sudah disetujui oleh Meta. Ini adalah aturan yang tidak bisa diabaikan.

Meta sudah menyediakan satu template bawaan bernama hello_world yang langsung bisa dipakai untuk testing tanpa perlu membuat template sendiri. Jika kamu hanya ingin test dulu, langsung lanjut ke Step 8.

Cara membuat template custom

Jika kamu ingin membuat template dengan isi pesan sendiri:

  1. Di menu kiri dashboard, klik WhatsApp lalu Message Templates
  2. Klik Create Template
  3. Pilih kategori Utility (paling cepat disetujui)
  4. Beri nama template, contoh: server_alert
  5. Pilih bahasa: Indonesian
  6. Tulis isi pesan di bagian Body. Gunakan {{1}}, {{2}}, dan seterusnya sebagai variabel yang akan diisi saat mengirim pesan
  7. Klik Submit

Status template akan berubah menjadi "Pending" dan biasanya disetujui dalam hitungan menit hingga 24 jam. Setelah statusnya "Approved", template siap digunakan.

Contoh isi template:

server_alert
{{1}} Alert — Server: {{2}} — Masalah: {{3}} — Level: {{4}} — Waktu: {{5}}

8 Kirim pesan pertama (testing)

Ini adalah momen yang ditunggu. Ada dua cara untuk mengirim pesan pertama.

Cara 1: Lewat dashboard Meta (tanpa koding)

Cara paling mudah, tidak perlu menulis kode apapun:

  1. Buka halaman API Setup di dashboard WhatsApp
  2. Di bagian From, pilih nomor pengirim (test number dari Meta)
  3. Di bagian To, pilih nomor penerima yang sudah didaftarkan
  4. Klik tombol Send Message
  5. Buka WhatsApp di HP penerima — pesan "Hello World" akan muncul

Cara 2: Lewat command line (cURL)

Ganti PHONE_NUMBER_ID dengan Phone Number ID kamu, dan ACCESS_TOKEN dengan token dari dashboard.

Terminal (Linux / Mac)
curl -X POST \
  "https://graph.facebook.com/v21.0/PHONE_NUMBER_ID/messages" \
  -H "Authorization: Bearer ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "messaging_product": "whatsapp",
    "to": "6281234567890",
    "type": "template",
    "template": {
      "name": "hello_world",
      "language": { "code": "en_US" }
    }
  }'

Jika berhasil, response-nya akan terlihat seperti ini:

Response JSON
{
  "messaging_product": "whatsapp",
  "contacts": [{ "wa_id": "6281234567890" }],
  "messages": [{ "id": "wamid.HBgNNjI4..." }]
}

Jika ada field messages dengan id di dalamnya, berarti pesan berhasil terkirim. Buka WhatsApp di HP penerima untuk memastikan.

9 Kirim pesan via Node.js

Untuk mengotomasi pengiriman pesan, buat project Node.js sederhana.

Buat project baru

Terminal
mkdir wa-bot
cd wa-bot
npm init -y
npm install dotenv

Buat file konfigurasi

Buat file bernama .env di dalam folder project. File ini menyimpan credentials yang bersifat rahasia:

.env
WA_PHONE_NUMBER_ID=123456789012345
WA_ACCESS_TOKEN=EAAGk0B3axxxxxxxxxx

Buat script pengirim

Buat file bernama send.js. Ganti nomor tujuan di baris paling bawah dengan nomor penerima kamu:

send.js
require('dotenv').config();
const https = require('https');

const PHONE_ID = process.env.WA_PHONE_NUMBER_ID;
const TOKEN = process.env.WA_ACCESS_TOKEN;

function kirim(nomor, template, lang) {
  return new Promise((resolve, reject) => {
    const body = JSON.stringify({
      messaging_product: 'whatsapp',
      to: nomor,
      type: 'template',
      template: {
        name: template,
        language: { code: lang || 'en_US' }
      }
    });
    const req = https.request({
      hostname: 'graph.facebook.com',
      path: '/v21.0/' + PHONE_ID + '/messages',
      method: 'POST',
      headers: {
        'Authorization': 'Bearer ' + TOKEN,
        'Content-Type': 'application/json'
      }
    }, res => {
      let data = '';
      res.on('data', c => data += c);
      res.on('end', () => {
        const json = JSON.parse(data);
        if (res.statusCode === 200) {
          console.log('Berhasil! Message ID:', json.messages[0].id);
          resolve(json);
        } else {
          console.error('Gagal:', json.error?.message);
          reject(new Error(json.error?.message));
        }
      });
    });
    req.on('error', reject);
    req.write(body);
    req.end();
  });
}

kirim('6281234567890', 'hello_world');

Jalankan

Terminal
node send.js

Jika output menampilkan "Berhasil! Message ID: ..." berarti pesan sudah terkirim. Buka WhatsApp penerima untuk memverifikasi.

10 Buat token yang tidak expire

Token yang didapat dari dashboard hanya berlaku 24 jam. Untuk penggunaan production, buat System User Token yang berlaku selamanya.

Buat System User

Buka business.facebook.com/settings/system-users. Klik tombol Add, beri nama (contoh: wa-bot), pilih role Admin, lalu klik Create System User.

Hubungkan dengan App

Klik nama system user yang baru dibuat, lalu klik Add Assets. Pilih tab Apps, centang app WhatsApp kamu, beri akses Full Control, lalu klik Save Changes.

Generate token

Masih di halaman system user, klik Generate New Token. Pilih app kamu, lalu centang dua permission berikut:

  • whatsapp_business_messaging
  • whatsapp_business_management

Klik Generate Token. Salin token yang muncul dan simpan di file .env. Token ini tidak punya masa berlaku.

Jaga kerahasiaan token Token ini setara dengan password. Jangan pernah membagikannya atau menyimpannya di repository publik. Pastikan file .env sudah dimasukkan ke .gitignore.

12 Opsional: integrasi dengan Zabbix

Jika kamu menggunakan Zabbix untuk monitoring server, berikut cara menghubungkan alert Zabbix ke WhatsApp Business API. Section ini bersifat opsional.

Yang perlu disiapkan

Sebelum integrasi, pastikan kamu sudah punya nilai-nilai berikut dari langkah sebelumnya:

CredentialDari manaContoh nilai
Phone Number IDDashboard Meta → API Setup (Step 5)123456789012345
Access Token (permanen)System User Token (Step 10)EAAGk0B3aZC...xxx
Nama templateMessage Templates (Step 7)server_alert
Nomor penerimaNomor WA tim NOC/admin6281234567890

Cara kerja integrasi

Alurnya sederhana: Zabbix mendeteksi masalah pada server, lalu memanggil script yang mengirim pesan WhatsApp ke tim terkait melalui Meta Cloud API. Berikut diagram alurnya:

  1. Zabbix Server mendeteksi problem (misal: CPU usage tinggi, disk penuh, service down)
  2. Zabbix menjalankan script alert yang sudah dikonfigurasi sebagai Media Type
  3. Script tersebut memanggil Meta Cloud API dengan credentials yang sudah disimpan
  4. Meta mengirimkan pesan template ke nomor penerima via WhatsApp
  5. Tim menerima notifikasi di HP masing-masing

File konfigurasi di server

Buat file .env di server tempat script berjalan. File ini berisi semua credentials yang dibutuhkan:

.env (di server monitoring)
# WhatsApp Business API
WA_PHONE_NUMBER_ID=123456789012345
WA_ACCESS_TOKEN=EAAGk0B3aZCxxxxx
WA_API_VERSION=v21.0

# Template yang sudah di-approve
WA_TEMPLATE_ALERT=server_alert
WA_TEMPLATE_RESOLVED=server_resolved

# Nomor penerima (pisah koma jika lebih dari satu)
WA_RECIPIENTS=6281234567890,6289876543210

Script integrasi

Buat file bernama zabbix-wa.js yang akan dipanggil oleh Zabbix setiap kali ada alert:

zabbix-wa.js
require('dotenv').config();
const https = require('https');

const PHONE_ID = process.env.WA_PHONE_NUMBER_ID;
const TOKEN = process.env.WA_ACCESS_TOKEN;
const VERSION = process.env.WA_API_VERSION || 'v21.0';

function kirimTemplate(nomor, template, params, lang) {
  return new Promise((resolve, reject) => {
    const body = JSON.stringify({
      messaging_product: 'whatsapp',
      to: nomor,
      type: 'template',
      template: {
        name: template,
        language: { code: lang || 'id' },
        components: params.length ? [{
          type: 'body',
          parameters: params.map(p => ({ type: 'text', text: String(p) }))
        }] : []
      }
    });
    const req = https.request({
      hostname: 'graph.facebook.com',
      path: '/' + VERSION + '/' + PHONE_ID + '/messages',
      method: 'POST',
      headers: {
        'Authorization': 'Bearer ' + TOKEN,
        'Content-Type': 'application/json'
      }
    }, res => {
      let data = '';
      res.on('data', c => data += c);
      res.on('end', () => {
        const json = JSON.parse(data);
        if (res.statusCode === 200) resolve(json);
        else reject(new Error(json.error?.message));
      });
    });
    req.on('error', reject);
    req.write(body);
    req.end();
  });
}

// Argumen dari Zabbix: node zabbix-wa.js "PROBLEM" "hostname" "problem desc" "severity"
const [,, status, host, problem, severity] = process.argv;
const recipients = (process.env.WA_RECIPIENTS || '').split(',').filter(Boolean);
const waktu = new Date().toLocaleString('id-ID');

(async () => {
  const template = status === 'OK'
    ? process.env.WA_TEMPLATE_RESOLVED
    : process.env.WA_TEMPLATE_ALERT;
  const params = status === 'OK'
    ? [host, problem, waktu]
    : [severity || 'WARNING', host, problem, 'Level ' + (severity || '3'), waktu];

  for (const nomor of recipients) {
    try {
      await kirimTemplate(nomor.trim(), template, params);
      console.log('Terkirim ke ' + nomor.trim());
    } catch (e) {
      console.error('Gagal ke ' + nomor.trim() + ': ' + e.message);
    }
  }
})();

Konfigurasi di Zabbix

Setelah script siap, hubungkan ke Zabbix:

  1. Buka Zabbix web interface
  2. Masuk ke Administration → Media Types
  3. Klik Create Media Type
  4. Isi:
    • Name: WhatsApp Business API
    • Type: Script
    • Script name: zabbix-wa.sh (wrapper yang memanggil node)
    • Script parameters: {ALERT.STATUS}, {HOST.NAME}, {ALERT.MESSAGE}, {EVENT.SEVERITY}
  5. Buat file wrapper /usr/lib/zabbix/alertscripts/zabbix-wa.sh:
zabbix-wa.sh
#!/bin/bash
cd /path/to/project
node zabbix-wa.js "$1" "$2" "$3" "$4"
  1. Beri permission: chmod +x /usr/lib/zabbix/alertscripts/zabbix-wa.sh
  2. Di menu Users, tambahkan Media Type "WhatsApp Business API" ke user admin
  3. Buat Action di Configuration → Actions untuk trigger notifikasi saat problem terjadi
Ringkasan credential yang perlu dicopy Hanya ada 4 nilai yang perlu disalin ke file .env di server: Phone Number ID, Access Token permanen, nama template, dan nomor penerima. Semua nilai ini didapat dari langkah-langkah sebelumnya di tutorial ini.
