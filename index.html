1 Apa itu WhatsApp Business API

Layanan resmi dari Meta untuk mengirim pesan WhatsApp secara otomatis melalui kode program.

Library Tidak Resmi (Baileys dll)

  • Bisa dibanned kapan saja
  • Perlu scan QR berulang
  • Sesi sering disconnect
  • Tidak ada support

WhatsApp Business API (Resmi)

  • Tidak bisa dibanned
  • Login pakai token, tanpa QR
  • 99.9% uptime
  • Dokumentasi lengkap dari Meta
Keterbatasan WA Business API hanya bisa kirim pesan personal (1-on-1). Tidak bisa kirim ke grup dan tidak bisa tag all.

3 Daftar akun Meta Developer

Langkah pertama adalah membuat akun developer dan app di Meta.

Buka Meta for Developers

Kunjungi developers.facebook.com dan login menggunakan akun Facebook.

Buat App Baru

Klik My Apps → Create App → pilih Other → pilih Business.

Isi nama app (contoh: WA Alert Bot), email, lalu klik Create App.

Buat Meta Business Account

Jika belum punya, buka business.facebook.com → Create Account. Isi nama bisnis, nama kamu, dan email.

4 Setup WhatsApp di dashboard

Tambahkan WhatsApp sebagai produk di app kamu dan catat credentials.

Tambahkan WhatsApp Product

Di dashboard app, scroll ke bawah, cari WhatsApp, klik Set Up. Pilih Business Account → Continue.

Catat 3 info penting

Di halaman API Setup, catat:

Credentials
Phone Number ID : 123456789012345
WABA ID         : 987654321098765
Access Token    : EAAGk0B3a...  (berlaku 24 jam)

Daftarkan nomor penerima

Selama test mode, kamu hanya bisa kirim ke nomor yang terdaftar:

  1. Di API Setup, klik Manage phone number list
  2. Klik Add phone number
  3. Masukkan nomor WA penerima: 6281234567890
  4. Penerima memasukkan kode verifikasi yang diterima
Format nomor Gunakan kode negara tanpa tanda plus. Contoh: 6281234567890 bukan +6281234567890 atau 081234567890.

5 Buat message template

Pesan harus dikirim dalam format template yang sudah di-approve Meta.

Kabar baik Meta sudah menyediakan template bawaan hello_world yang langsung bisa dipakai untuk testing. Kamu bisa langsung ke Step 6.

Membuat template custom (opsional)

  1. Buka WhatsApp Manager → Message Templates
  2. Klik Create Template
  3. Pilih kategori Utility, beri nama, pilih bahasa Indonesian
  4. Tulis body template. Gunakan {{1}}, {{2}} dst sebagai variabel
  5. Submit dan tunggu approval (biasanya beberapa menit sampai 24 jam)

Contoh template:

server_alert (Utility)
{{1}} Alert
Server: {{2}}
Masalah: {{3}}
Level: {{4}}
Waktu: {{5}}

6 Kirim pesan pertama

Sekarang saatnya test. Ada 2 cara: via dashboard atau via command line.

Cara 1: Via Dashboard (paling mudah)

  1. Di WhatsApp → API Setup
  2. Pilih nomor pengirim (From) dan nomor penerima (To)
  3. Klik Send Message
  4. Cek HP penerima — pesan "Hello World" masuk

Cara 2: Via cURL

Ganti PHONE_NUMBER_ID dan ACCESS_TOKEN dengan milik kamu.

Linux / Mac
curl -X POST \
  "https://graph.facebook.com/v21.0/PHONE_NUMBER_ID/messages" \
  -H "Authorization: Bearer ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "messaging_product": "whatsapp",
    "to": "6281234567890",
    "type": "template",
    "template": {
      "name": "hello_world",
      "language": { "code": "en_US" }
    }
  }'

Response sukses:

JSON Response
{
  "messaging_product": "whatsapp",
  "contacts": [{ "wa_id": "6281234567890" }],
  "messages": [{ "id": "wamid.HBgN..." }]
}

Jika muncul field messages dengan id, berarti pesan berhasil terkirim. Cek HP penerima.

7 Kirim via Node.js

Untuk otomasi, kirim pesan menggunakan Node.js.

Setup project

Terminal
mkdir wa-bot && cd wa-bot
npm init -y
npm install dotenv

File .env

.env
WA_PHONE_NUMBER_ID=123456789012345
WA_ACCESS_TOKEN=EAAGk0B3axxxxxxxxxx

File send.js

send.js
require('dotenv').config();
const https = require('https');

const PHONE_ID = process.env.WA_PHONE_NUMBER_ID;
const TOKEN = process.env.WA_ACCESS_TOKEN;

function kirim(nomor, template, params = [], lang = 'en_US') {
  return new Promise((resolve, reject) => {
    const body = JSON.stringify({
      messaging_product: 'whatsapp',
      to: nomor,
      type: 'template',
      template: {
        name: template,
        language: { code: lang },
        components: params.length ? [{
          type: 'body',
          parameters: params.map(p => ({ type: 'text', text: String(p) }))
        }] : []
      }
    });

    const req = https.request({
      hostname: 'graph.facebook.com',
      path: '/v21.0/' + PHONE_ID + '/messages',
      method: 'POST',
      headers: {
        'Authorization': 'Bearer ' + TOKEN,
        'Content-Type': 'application/json'
      }
    }, res => {
      let data = '';
      res.on('data', c => data += c);
      res.on('end', () => {
        const json = JSON.parse(data);
        if (res.statusCode === 200) {
          console.log('Pesan terkirim! ID:', json.messages[0].id);
          resolve(json);
        } else {
          console.error('Gagal:', json.error?.message);
          reject(new Error(json.error?.message));
        }
      });
    });
    req.on('error', reject);
    req.write(body);
    req.end();
  });
}

// Ganti nomor di bawah dengan nomor tujuan
kirim('6281234567890', 'hello_world');

Jalankan

Terminal
node send.js

# Output:
# Pesan terkirim! ID: wamid.HBgNNjI4...

8 Buat token permanen

Token dari dashboard hanya berlaku 24 jam. Untuk production, buat System User Token.

Buat System User

Buka Business Settings → System Users. Klik Add, beri nama wa-bot, role Admin.

Assign asset

Klik user → Add Assets → pilih tab Apps → centang app WA kamu → Full Control → Save.

Generate token

Klik Generate New Token → pilih app → centang whatsapp_business_messaging dan whatsapp_business_management → Generate Token.

Copy token yang muncul dan simpan di file .env. Token ini tidak expire.

Penting Token ini seperti password. Jangan commit ke GitHub. Masukkan .env ke .gitignore.
